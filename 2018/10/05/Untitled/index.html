Hexo+github+Hexo editor搭建自己的博客（Mac）

Posted on 2018-10-05 | Edited on 2018-10-09

#分享是提高认知的唯一途径#

相信很多技术咖一直都有很多idea，但总会选择拖延式实现，慢慢的，最后选择让这些idea消磨殆尽。有人说，灵感需要捕抓，有思想的人或者更充实吧。说干就干，我今天给大家分享一下自己搭建Hexo的全过程。

1.用Hexo在本地搭建一个博客

1.1 下载并安装nodejs

1.2 安装git

1.3 打开终端使用npm安装hexo

  • sudo -i //切换root权限
  • npm install -g hexo-cli
  • //等待success

快速开始

#在终端输入

  • hexo i blog //i是init的缩写，blog是你的博客项目名，你可以自己自定义
  • cd blog //切换到站点根目录
  • hexo g //g是generetor的缩写，生成默认静态文件
  • hexo s //s是server的缩写，部署代码到本地

在浏览器打开输入 localhost:4000 即可查看

2.Hexo主题选择——next

2.1 在站点根目录输入：

  • git clone https://github.com/theme-next/hexo-theme-next themes/next

2.2 完成后，打开站点的配置文件，走到theme字段，并将其值改为 next

2.3 在终端输入

  • hexo clean //清除缓存
  • hexo g //重新生成代码
  • hexo s //部署到本地

//在浏览器打开输入 localhost:4000 查看效果

next主题有三种，你可自己选择切换使用：

  • Muse - 默认 Scheme，这是 NexT 最初的版本，黑白主调，大量留白
  • Mist - Muse 的紧凑版本，整洁有序的单栏外观
  • Pisces - 双栏 Scheme，小家碧玉似的清新

3.5 通过SSH方式将hexo项目提交到github

#检查本机是否存在ssh公钥

  • cd ~./ssh

#创建新的SSH密钥对

  • ssh-keygen -t rsa -C”your_email@example.com“
  • Enter passphrase (empty for no passphrase):[输入密码]
  • Enter same passphrase again: [再次输入密码]

#查看或编辑密钥文件

  • cat id_rsa.pub //查看密钥文件
  • vim id_rsa.pub //编辑密钥文件

#将密钥添加到github上



#测试是否连接成功

  • ssh -T git@github.com

4. Hexo-editor一款简洁的Hexo Web编辑器

#获取编辑器

  • git clone https://github.com/tajpure/hexo-editor.git
    #初始化
  • cd hexo-editor
  • npm install –production

#base_dir（绝对路径）//我的绝对路径示例：Users/evn/pm
将您的hexo博客目录设置为“base_dir”。

#如果需要部署hexo博客，则应将SSH密钥添加到目标计算机的authorized_keys。

#Hexo Editor的默认端口是2048

#开始
npm start //在hexo-editor路径下执行

如果你是部署到本地的，访问：locallhost:2048

文档地址：https://github.com/tajpure/hexo-editor/wiki
